Welcome to Logos

Setup connects your models, benchmarks them, configures where agents run and how safely they're sandboxed, and secures your account. These choices define the platform’s initial profile. Everything here can be adjusted from the dashboard after launch.

Connect your model, run a quick benchmark, pick an agent, set a soul, create an account. Everything can be changed later.

Each agent run is recorded as a STAMP — Soul + Tools + Agent + Model + Policy — making sessions reproducible, comparable, and auditable. Setup defines the defaults that every STAMP inherits.

Every session is logged so you can replay or compare runs later.

You’ll have a fully configured platform: inference routed to a benchmarked local model, agent runs sandboxed with policy-enforced isolation (or in-process if Docker isn’t available), a soul selected, and a secured admin account. The platform is ready for agent runs.

Running on a VPC, cloud VM, or remote server

You only need to install the inference software on the remote machine — not Logos.

Ollama on a Linux VM

curl -fsSL https://ollama.com/install.sh | sh

Then expose the API publicly (binds to all interfaces):

OLLAMA_HOST=0.0.0.0 ollama serve

Then pull your model: ollama pull qwen2.5:14b

vLLM on a GPU VM

pip install vllm
vllm serve Qwen/Qwen2.5-14B-Instruct --host 0.0.0.0 --port 8000

Networking checklist

  • Open the inference port in the VM's firewall / security group (e.g. TCP 11434 for Ollama, 8000 for vLLM)
  • If behind a private VPC with no public IP, use a VPN, SSH tunnel, or reverse proxy
  • On AWS/GCP/Azure: add an inbound rule in the security group / firewall for the port

Quick SSH tunnel (no firewall changes needed)

ssh -L 11434:localhost:11434 user@your-vm-ip

Then add http://localhost:11434 as your server URL above — traffic tunnels securely.

What do these scores mean?
tok/s — tokens per second: how fast the model generates text. Higher is better. Agent conversations feel sluggish below ~6 tok/s. ≥30 fast 15–29 good 6–14 usable <6 slow
ctx — context window: the maximum number of tokens the model can hold at once (your message + its reply + everything in between). The agent system prompt alone uses ~8K tokens, so a model with an 8K context window has almost no room left for your conversation. Models under 16K ctx are not eligible for recommendation — they will show a ⚠ and cannot be selected as best. If your model only supports 8K, use the General (Lite) soul which has a smaller footprint.
✓ / ⚠ — whether the model passed the quality eval (instruction following, reasoning, structured output, tool selection). Models that fail the format or tool-call tests are de-prioritised because they will break agent loops.
Results depend on current server load. For accurate scores, avoid running other workloads on your inference machines during the benchmark.
Scores reflect conditions at test time. A server handling other requests or services during testing may show lower throughput than usual.
